## Обзор

Реализация российского криптостека (ГОСТ) в виде подключаемого провайдера для HTTPS VPN. Включает в себя низкоуровневые примитивы и кастомную реализацию TLS, совместимую с RFC 9189.

## Затронутые системы

| Система | Влияние | Примечания |
|--------|--------|-------|
| `crypto/ru/gost` | Создание | Реализация алгоритмов Кузнечик, Магма, Стрибог, ГОСТ 34.10 |
| `crypto/ru/tls` | Создание | Кастомный стек TLS для поддержки ГОСТ cipher suites |
| `crypto/provider.go` | Изменение | Интеграция провайдера "ru" |

## Архитектура

### Компоненты

```
[transport/server] ──> [crypto.Provider ("ru")]
                             │
                             └──> [crypto/ru/tls]
                                       │
                                       └──> [crypto/ru/gost]
```

## Интерфейсы

### Провайдер "ru"
Реализует интерфейс `crypto.Provider`:
- `Name() string` -> "ru"
- `ConfigureTLS(cfg *tls.Config) error`
- `SupportedCipherSuites() []uint16`

## Модели данных

### Поддерживаемые Cipher Suites (RFC 9189)
- `0xC100`: TLS_GOSTR341112_256_WITH_KUZNYECHIK_CTR_OMAC
- `0xC101`: TLS_GOSTR341112_256_WITH_KUZNYECHIK_MGM_L
- `0xC103`: TLS_GOSTR341112_256_WITH_MAGMA_CTR_OMAC

## Спецификации поведения

### Установление соединения (Happy Path)
1. Сервер считывает конфиг с `cipherSuites: "ru"`.
2. Загружается провайдер ГОСТ.
3. При подключении клиента сервер предлагает наборы шифров `0xC1xx`.
4. Выполняется VKO обмен ключами.
5. Данные шифруются с использованием MGM или CTR_OMAC.

## Тестирование

- **Unit-тесты**: Проверка примитивов на тестовых векторах ГОСТ Р 34.12 и 34.11.
- **Интеграционные тесты**: Эмуляция полного цикла TLS Handshake с использованием ГОСТ сертификатов.
