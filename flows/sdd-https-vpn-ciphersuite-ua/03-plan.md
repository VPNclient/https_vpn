# План імплементації: Українська постквантова криптографія (ДСТУ-ПК 2026)

> Версія: 1.0
> Статус: APPROVED
> Останнє оновлення: 2026-04-27
> Специфікації: [02-specifications.md](./02-specifications.md)

## Огляд

План реалізації UA криптографічного провайдера розбитий на 4 фази: базові примітиви, TLS інтеграція, системна інтеграція та тестування.

## Розбивка завдань

### Фаза 1: Базові криптографічні примітиви

#### Завдання 1.1: Хеш-функція Купина-512

- **Опис**: Реалізація хеш-функції ДСТУ 7564:2014 з 512-бітним виходом
- **Файли**:
  - `crypto/ua/kupyna/kupyna.go` - Створити
  - `crypto/ua/kupyna/kupyna_test.go` - Створити
  - `crypto/ua/kupyna/tables.go` - Створити (S-box та MDS матриці)
- **Залежності**: Немає
- **Верифікація**: Тести з офіційними тестовими векторами ДСТУ 7564
- **Складність**: Середня

#### Завдання 1.2: Блочний шифр Калина-512

- **Опис**: Реалізація блочного шифру ДСТУ 7624:2014 з 512-бітним ключем
- **Файли**:
  - `crypto/ua/kalyna/kalyna.go` - Створити
  - `crypto/ua/kalyna/kalyna_test.go` - Створити
  - `crypto/ua/kalyna/tables.go` - Створити
  - `crypto/ua/kalyna/gcm.go` - Створити (GCM режим)
  - `crypto/ua/kalyna/gcm_test.go` - Створити
- **Залежності**: Завдання 1.1 (Купина для GCM/GHASH)
- **Верифікація**: Тести з офіційними тестовими векторами ДСТУ 7624
- **Складність**: Висока

#### Завдання 1.3: KEM Мальва (Malva)

- **Опис**: Реалізація Key Encapsulation Mechanism на базі Module-LWE
- **Файли**:
  - `crypto/ua/malva/malva.go` - Створити
  - `crypto/ua/malva/malva_test.go` - Створити
  - `crypto/ua/malva/ntt.go` - Створити (Number Theoretic Transform)
  - `crypto/ua/malva/params.go` - Створити (параметри аналогічні Kyber-1024)
  - `crypto/ua/malva/poly.go` - Створити (операції з поліномами)
- **Залежності**: Завдання 1.1 (Купина для хешування)
- **Верифікація**: Тести KAT (Known Answer Tests)
- **Складність**: Висока

#### Завдання 1.4: Цифровий підпис Сокіл (Sokil)

- **Опис**: Реалізація цифрового підпису на базі SIS/решіток
- **Файли**:
  - `crypto/ua/sokil/sokil.go` - Створити
  - `crypto/ua/sokil/sokil_test.go` - Створити
  - `crypto/ua/sokil/ntt.go` - Створити
  - `crypto/ua/sokil/params.go` - Створити (параметри аналогічні Dilithium-5)
  - `crypto/ua/sokil/poly.go` - Створити
  - `crypto/ua/sokil/sampling.go` - Створити (вибірка з розподілів)
- **Залежності**: Завдання 1.1 (Купина для хешування)
- **Верифікація**: Тести KAT, тести верифікації підпису
- **Складність**: Висока

### Фаза 2: TLS інтеграція

#### Завдання 2.1: TLS константи та типи

- **Опис**: Визначення cipher suite IDs та curve IDs
- **Файли**:
  - `crypto/ua/tls/cipher_suites.go` - Створити
  - `crypto/ua/tls/cipher_suites_test.go` - Створити
- **Залежності**: Немає
- **Верифікація**: Компіляція без помилок, тести ID унікальності
- **Складність**: Низька

#### Завдання 2.2: UA провайдер

- **Опис**: Реалізація інтерфейсу crypto.Provider
- **Файли**:
  - `crypto/ua/provider.go` - Створити
  - `crypto/ua/provider_test.go` - Створити
- **Залежності**: Завдання 1.1-1.4, 2.1
- **Верифікація**: Тести реєстрації та конфігурації TLS
- **Складність**: Середня

#### Завдання 2.3: Гібридний KEM

- **Опис**: Комбінація X25519 + Malva для key exchange
- **Файли**:
  - `crypto/ua/tls/hybrid_kem.go` - Створити
  - `crypto/ua/tls/hybrid_kem_test.go` - Створити
- **Залежності**: Завдання 1.3
- **Верифікація**: Тести з X25519 + Malva encapsulation/decapsulation
- **Складність**: Середня

### Фаза 3: Системна інтеграція

#### Завдання 3.1: Детекція UA криптографії

- **Опис**: Додати IsUACryptoSuite() до базового провайдера
- **Файли**:
  - `crypto/provider.go` - Модифікувати
  - `crypto/crypto_test.go` - Модифікувати
- **Залежності**: Завдання 2.1
- **Верифікація**: Тести детекції UA suites
- **Складність**: Низька

#### Завдання 3.2: CertificateStore інтеграція

- **Опис**: Автовибір UA сертифікатів за типом ключа
- **Файли**:
  - `crypto/certstore.go` - Модифікувати
  - `crypto/certstore_test.go` - Модифікувати
- **Залежності**: Завдання 1.4, 3.1
- **Верифікація**: Тести автовибору сертифіката з Sokil ключем
- **Складність**: Середня

#### Завдання 3.3: Конфігурація

- **Опис**: Приклад конфігурації для UA провайдера
- **Файли**:

- **Залежності**: Завдання 2.2
- **Верифікація**: Сервер стартує з UA конфігурацією
- **Складність**: Низька

## Стратегія відкату

Якщо імплементація не вдається або потребує скасування:

1. Видалити каталог `crypto/ua/`
2. Відкатити зміни в `crypto/provider.go`
3. Відкатити зміни в `crypto/certstore.go`
4. Видалити `config.ua.json`
