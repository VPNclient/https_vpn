# Российская криптография (ГОСТ) для HTTPS VPN

Этот каталог содержит реализацию криптографических примитивов и TLS-логики на базе российских стандартов **ГОСТ** (в контексте проекта — “ГОСТ VPN”: TLS/канал управления и туннель поверх HTTPS с национальными алгоритмами).

## Что реализовано в коде

- **Криптографические примитивы**: `crypto/ru/gost/`
  - **ГОСТ Р 34.11-2012 (Стрибог)**: `streebog.go`
  - **ГОСТ Р 34.10-2012 (ЭЦП, кривые, ключи)**: `gost3410.go`, `curves.go`
  - **ГОСТ Р 34.12-2015 (Кузнечик/Магма)**: `kuznyechik.go`, `magma.go`
  - **Режимы/имитовставка** (используются в TLS-наборах): `ctr.go`, `omac.go`, `mgm.go`, `hmac.go`

- **TLS (ГОСТ cipher suites / handshake)**: `crypto/ru/tls/`
  - Константы и метаданные наборов шифров: `cipher_suites.go`
  - Логика рукопожатия и записи: `handshake.go`, `record.go`
  - VKO (выработка общего секрета для ГОСТ-обмена): `vko.go`

## TLS cipher suites (ГОСТ)

В `crypto/ru/tls/cipher_suites.go` описаны ГОСТ-наборы шифров (включая варианты с **Кузнечиком** и **Магмой**, а также хешем **Стрибог-256/512**) с идентификаторами:

- `0xC100` … `0xC10B` — ГОСТ-наборы шифров (см. исходники для точного соответствия “ID → имя/параметры”)

Проект также содержит список “поддерживаемых” наборов в `SupportedCipherSuites()` — это полезно как ориентир того, какие suites считаются рабочими в текущей реализации.

## Как включить “ГОСТ VPN” (сервер)

В конфигурации используйте выбор провайдера через поле `tlsSettings.cipherSuites`:

```json
{
  "tlsSettings": {
    "cipherSuites": "ru"
  }
}
```

Пояснение: в этом проекте строковое поле `cipherSuites` используется **двойным образом** — как список стандартных suites и как выбор национального крипто-провайдера (например, `"ru"`, `"cn"`, `"us"`). Спецификация: `flows/sdd-vpn-https-config-ciphersuites/02-specifications.md`.

## Программное использование (провайдер)

Реестр провайдеров находится в `crypto/` (см. `crypto/provider.go`). Конкретный ГОСТ-провайдер регистрируется при импорте соответствующего пакета “ru”.

Минимальный шаблон выглядит так:

```go
import (
	"github.com/vpnclient/https-vpn/crypto"
	_ "github.com/vpnclient/https-vpn/crypto/ru" // регистрация провайдера "ru" (если пакет провайдера присутствует в сборке)
)

provider, ok := crypto.Get("ru")
_ = provider
_ = ok
```

## Примечания по совместимости

- **Клиентская совместимость**: для установления TLS-сессии с ГОСТ наборами шифров требуется клиент, умеющий в ГОСТ (кривые/подписи/наборы).
- **Сертификаты**: для “чистого” ГОСТ режима обычно нужны сертификаты с ГОСТ-ключами/подписями (ГОСТ Р 34.10-2012). В проекте есть инфраструктура выбора сертификата по провайдеру (см. `crypto/certstore.go`).
